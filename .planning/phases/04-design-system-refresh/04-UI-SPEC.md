## Token Changes: Locked Decisions

These changes are **mandated** by the design brief. Executor must apply them in `globals.css`.

### Accent color replacement

| Token | Before | After | Rationale |
|-------|--------|-------|-----------|
| `--color-dr-glow` | `#5B87FF` | `#00D97E` | Emerald replaces blue as primary accent |
| `--color-dr-glow-dim` | `#5B87FF1A` | `#00D97E1A` | Dim variant follows main glow |
| `--color-dr-glow-mid` | `#5B87FF44` | `#00D97E44` | Mid variant follows main glow |
| `--color-dr-status-active` | `#5B87FF` | `#00D97E` | Active status tag uses new accent |
| `--color-dr-role-frontend` | `#5B87FF` | `#00D97E` | Frontend role chip uses new accent |
| `--color-dr-axis-initiative` | `#5B87FF` | `#00D97E` | Soft-skill initiative axis uses new accent |

Add new tokens for emerald alias (used by future phases — radar chart fill, empty state tints):

```css
--color-dr-emerald:     #00D97E;
--color-dr-emerald-dim: #00D97E1A;
--color-dr-emerald-mid: #00D97E44;
--color-dr-info:        #5B87FF;   /* blue demoted to info/secondary role */
```

**Aurora gradients in `.dr-aurora`:** replace `rgba(91, 135, 255, …)` with `rgba(0, 217, 126, …)` at same opacity values.

**`.dr-glow-ring` box-shadow:** replace `rgba(91, 135, 255, …)` with `rgba(0, 217, 126, …)` at same opacity values.

**`.dr-card:hover` box-shadow:** replace `rgba(91, 135, 255, 0.07)` with `rgba(0, 217, 126, 0.07)`.

**`.dr-btn-primary` gradient:** replace `#5B87FF → #3B65D8` with `#00D97E → #00A85F`.

**`.dr-text-glow` gradient:** replace `#B0CCFF / #5B87FF / #3B65D8` with `#8FFFC9 / #00D97E / #00A85F`.

**`::selection`:** replace `rgba(91, 135, 255, 0.22)` with `rgba(0, 217, 126, 0.22)`.

**Hero profile avatar** (hero/me/page.tsx): uses `--color-dr-glow-dim` / `--color-dr-glow` — updates automatically via token.

**Filter chip "All" state** (raids/page.tsx): uses `var(--color-dr-glow-dim)` / `var(--color-dr-glow)` — updates automatically.

**Hard skills chips**: use `var(--color-dr-glow)` / `var(--color-dr-glow-dim)` / `var(--color-dr-glow-mid)` — updates automatically.

### Micro-details

**Section divider with runic accent:**

Add `.dr-divider-rune` utility to `globals.css`:

```css
/* Section divider with centered diamond accent */
.dr-divider-rune {
  position: relative;
  height: 1px;
  border: none;
  background: linear-gradient(
    90deg,
    transparent 0%,
    rgba(255, 255, 255, 0.07) 20%,
    rgba(255, 255, 255, 0.07) 80%,
    transparent 100%
  );
  margin: 0;
}

.dr-divider-rune::after {
  content: '◆';
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  font-size: 8px;
  color: var(--color-dr-muted);
  background: var(--color-dr-bg);
  padding: 0 4px;
  line-height: 1;
}
```

Use `.dr-divider-rune` instead of `.dr-divider` where section separators appear between named sections on a page. The plain `.dr-divider` remains for tight functional separators (table rows, form sections).

**Animated gradient border on interactive `.dr-card`:**

For cards that are interactive (wrapped in `<Link>` or have `onClick`), add a hover border shimmer via CSS custom property:

```css
.dr-card-interactive {
  background-image: none; /* baseline */
  transition:
    box-shadow  0.25s ease,
    border-color 0.25s ease,
    transform    0.25s ease;
}

.dr-card-interactive:hover {
  border-color: transparent;
  background-clip: padding-box;
  box-shadow:
    0 1px 0 0 rgba(255, 255, 255, 0.06) inset,
    0 8px 36px rgba(0, 0, 0, 0.65),
    0 0 0 1px rgba(0, 217, 126, 0.18);
}
```

This is additive — apply `.dr-card-interactive` alongside `.dr-card` on clickable cards. Existing `.dr-card` hover rules are unmodified.

### Empty states: SVG illustrations

SVG accents go **only** in empty/zero-data states. Must NOT appear when data is present.

Empty state locations in scope:
1. `app/raids/page.tsx` — when raid list is empty
2. `app/hero/me/page.tsx` — when `hard_skills` array is empty

Empty state anatomy:
- SVG illustration: inline, 80–120px tall, `aria-hidden="true"`, decorative only
- Heading: 16px Sora weight 600, `--color-dr-text`
- Body: 14px Sora weight 400, `--color-dr-text-2`
- CTA if actionable: `.dr-btn-secondary` or plain link

See Copywriting Contract below for exact copy.

## Spacing Scale

Standard 8-point scale. Existing codebase uses Tailwind utility classes (`p-6`, `gap-4`, etc.) which map to this scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding |
| sm | 8px | Compact badge/chip spacing |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding (current `p-6`) |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets for interactive buttons: minimum 44px height (existing `.dr-btn-primary` padding ~40px — acceptable at MVP scale)
- Diamond in `.dr-divider-rune::after`: 8px font-size, 4px horizontal padding — exception to 8-point rule, treated as a decorative micro-element

## Typography

Four size tiers. The `.dr-label` class is a **Body-tier variant** — it shares the 14px size but applies JetBrains Mono, uppercase transform, and tighter tracking. It does not constitute a separate size tier.

| Role | Size | Weight | Line Height | Font | Class / Usage |
|------|------|--------|-------------|------|---------------|
| Display | clamp(3.5rem, 9vw, 6rem) | 700 | 1.0 (leading-none) | Cinzel | Page hero titles (`.dr-heading`) |
| Heading | 20px (1.25rem) | 700 | 1.15 | Cinzel | h1/h2 section headers (`.dr-heading`) |
| Subheading | 16px (1rem) | 600 | 1.3 | Sora | h3, card titles |
| Body | 14px (0.875rem) | 400 | 1.5 | Sora | Paragraph text, form labels |

**Body-tier variant — `.dr-label`:** 14px, weight 400, line-height 1.4, JetBrains Mono, `text-transform: uppercase`, `letter-spacing: 0.08em`. Used for badges, meta text, and uppercase tags. Size is identical to Body; only font family and casing differ.

**Two weights in use:** 400 (regular) and 600/700 (semibold/bold). No intermediate weights.

**Cinzel activation rule:** `.dr-heading` applies Cinzel. Use only on `<h1>` and `<h2>`. Never on body copy, labels, or buttons.

## Interactions

### Hover states
- Cards (`.dr-card`): `translateY(-1px)` + border shift + emerald ring `rgba(0,217,126,0.07)` — 0.25s ease (existing, auto-updates with token)
- Interactive cards (`.dr-card-interactive`): emerald ring intensifies to `rgba(0,217,126,0.18)` on hover
- Buttons (`.dr-btn-primary`): `translateY(-1px)` + deeper emerald glow — 0.15s ease

### Animations
- Page entrance: `.dr-enter` through `.dr-enter-5` — `dr-fade-up` 0.55s cubic-bezier(0.4,0,0.2,1) with staggered delays (existing, keep)
- Aurora: `dr-aurora-drift` 24s infinite (existing, keep — update to emerald gradient)
- Edit panel: `AnimatePresence` height + opacity, 0.25s easeOut in / 0.18s out (existing, keep)
- `useReducedMotion`: already implemented across pages — all new animations must respect this pattern by spreading `{}` instead of animation props

### Motion contract for new empty state components
- On mount: `dr-enter` CSS animation (opacity 0→1, translateY 18px→0, 0.55s)
- No loop animations on SVG illustrations
- `prefers-reduced-motion`: render static, skip animation class

## Illustrative SVG Spec

SVGs are inline elements, not external images or `<img>` tags.

Requirements:
- Decorative only: `aria-hidden="true"`, no `role="img"`, no `<title>` needed
- Monochromatic or two-tone using existing tokens only
- Height: 80–120px, width: fluid (viewBox-based)
- Style: simple geometric / silhouette, fantasy-dark aesthetic
- No photographic references, no complex multi-stop gradients, no external references

**Raids empty state SVG:** Stylised crossed swords or shield silhouette.
- Body fill: `--color-dr-edge-2` (`#263047`)
- Accent detail stroke: `--color-dr-glow` (`#00D97E`)

**Hard skills empty state SVG:** Stylised scroll or rune tablet.
- Body fill: `--color-dr-edge-2` (`#263047`)
- Accent edge stroke: `--color-dr-gold` (`#C8943A`)

**Render rule:** SVG renders only when the data array length is 0. When data is present, the SVG does not render (conditional branch, not CSS display toggle).
