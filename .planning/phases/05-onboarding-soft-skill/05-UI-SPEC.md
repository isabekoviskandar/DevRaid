## Interaction & Animation

### Form State Management

**Current step:** React state `const [currentStep, setCurrentStep] = useState(0)`
**Answers:** Object `{ [questionId]: 'A'|'B'|'C'|'D' }`
**Progress:** Calculated as `(currentStep + 1) / 20`

### Step Transitions

**On "Next" button click:**
1. Validate answer is selected (disable button if not)
2. Save answer to state
3. Increment step counter
4. Trigger progress bar animation (width 0.4s ease)
5. Fade out current question card (optional, 0.15s)
6. Fade in new question card with stagger (0.15s delay → 0.25s animation)
7. Scroll to top of card (smooth, 0.3s)

**Animation timings (Framer Motion ready, but CSS fallback provided):**
- Progress bar: `0.4s cubic-bezier(0.4,0,0.2,1)`
- Card fade out: `0.15s ease`
- Card fade in: `0.25s ease` (delayed 0.15s)
- Total step transition: ~0.5s feel

**CSS Keyframe fallback (if Framer Motion not used):**
```css
@keyframes dr-fade-up {
  from { opacity: 0; transform: translateY(18px); }
  to   { opacity: 1; transform: translateY(0);    }
}

.dr-card-enter {
  animation: dr-fade-up 0.25s cubic-bezier(0.4,0,0.2,1) 0.15s both;
}
```

### Radio Button Interaction

**Click behavior:**
1. Select radio input (native)
2. Highlight card with glow ring (0.18s ease)
3. Enable "Next" button (if disabled)
4. Optional: play subtle sound effect (accessibility: reduce-motion respected)

**Keyboard navigation (native radio group):**
- `Tab` → focus first radio, move between radios
- `Arrow Up/Down` → cycle through options
- `Space` → select option
- `Enter` on button → submit

### Results Screen (Step 21)

**Trigger:** After step 20 submitted
**Flow:**
1. Show results page with hexagon preview (minimal, Phase 6 ready)
2. Display "Your Soft Skills Profile" heading
3. Show 5 values as numbers or small bars (no full chart yet)
4. CTA button: "Go to Profile" → `window.location.href = '/hero/me'` or `router.push('/hero/me')`

**Timing:** Results show immediately (0 API delay), hexagon values hardcoded from weighting algorithm in question weights.

## Keyboard & Accessibility

### Navigation Flow

1. **Tab order:** Skip → Progress bar (skip) → Step counter (skip) → Question heading (skip) → Answer A (focus) → Answer B → Answer C → Answer D → Next button → {next iteration}
2. **ARIA labels:**
   - `<form aria-label="Soft skills assessment form">`
   - `<fieldset><legend>Question 5 of 20: {question text}</legend>`
   - Progress bar: `<div role="progressbar" aria-valuenow="5" aria-valuemin="1" aria-valuemax="20" aria-label="Progress: 5 of 20 complete">`
   - Next button: `aria-label="Continue to question 6"` (optional, for context)

### Reduced Motion

Respect `prefers-reduced-motion` media query:
```css
@media (prefers-reduced-motion: reduce) {
  .dr-card-enter { animation: none; opacity: 1; }
  .dr-progress-bar { transition: width 0s linear; }
}
```
