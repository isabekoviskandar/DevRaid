# Phase 15 - UI Design Contract: Design System 2.0

> Visual and interaction contract for the Emerald Guild system layer only.
> Generated by gsd-ui-researcher. Intended for `gsd-planner`, `gsd-executor`, `gsd-ui-checker`, and `gsd-ui-auditor`.
> Sources: `.planning/STATE.md`, `.planning/ROADMAP.md`, `.planning/REQUIREMENTS.md`, `15-CONTEXT.md`, `15-RESEARCH.md`, `.planning/PROJECT.md`, `app/globals.css`, `app/layout.tsx`, `components/layout/app-shell.tsx`, auth and landing page audits.

---

## Scope

| Area               | Contract                                                                                                                                                          |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| In scope           | CSS token ownership, surface mood, background language, typography hierarchy, glow restraint, motion tokens, z-index ladder, additive utility contracts in `app/globals.css` |
| Out of scope       | Page redesigns, auth-page bespoke layouts, component rebuilds, particle FX, typewriter, sound, ScrollTrigger, new page-specific markup                         |
| File boundary      | `app/globals.css` is the only required implementation file for this phase                                                                                         |
| Compatibility rule | Existing `.dr-card`, `.dr-btn-*`, `.dr-heading`, `.dr-title-fantasy`, `.dr-glow-ring`, `.dr-card--*` contracts must continue to work unchanged                |

Phase 15 is a system-layer correction, not a page makeover. The old "Obsidian Citadel" bias is superseded. The system must now read as **Emerald Guild**: dark foundation, lighter emerald/taupe panels, mist and subtle hex-grid atmosphere, restrained premium glow, and fantasy typography only where hierarchy benefits from it.

---

## Design System

| Property          | Value                                                                                       |
| ----------------- | ------------------------------------------------------------------------------------------- |
| Tool              | none (custom CSS tokens via Tailwind v4 `@theme` plus runtime CSS variables)               |
| Preset            | not applicable                                                                              |
| Component library | Radix UI primitives plus bespoke `.dr-*` utility system                                     |
| Icon library      | Lucide React plus inline SVG hex/rune marks                                                 |
| Font              | Sora for body UI, JetBrains Mono for labels/meta, Cinzel for display and page-title moments |

`components.json` is absent, so no shadcn preset applies. The shadcn initialization gate is intentionally not used for this phase because the repo already has an active CSS-first design system, the phase is explicitly compatibility-safe for existing `.dr-*` contracts, and the approved scope is global token architecture rather than kit adoption.

## Reference Translation

| Reference                 | What it contributes to the contract                                                                 |
| ------------------------- | -------------------------------------------------------------------------------------------------- |
| Stitch emerald quest board | Misty emerald ambience, board-like panel layering, readable magical atmosphere, premium glow without neon overload |
| fantasy-taupe            | Taupe support tones, pointed/fantasy edge language, decorative frame accents, display-font flavor reserved for special hierarchy |
| boot.dev                 | Readable dark UI density, practical layout clarity, disciplined CTA emphasis, clean product-grade structure over ornament spam |

The executor must treat these references as a blend, not three parallel directions. `Stitch` sets atmosphere, `fantasy-taupe` sets edge treatment, and `boot.dev` sets readability discipline.

## Token Ownership

| Layer                       | Owner            | Tokens                                                                                                       | Contract                                                                                                   |
| --------------------------- | ---------------- | ------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------- |
| Utility-backed theme tokens | `@theme`         | `--color-dr-*`, `--font-*`, `--font-display`, `--text-*`, `--tracking-*`, `--radius-*`, `--shadow-*`, `--ease-*` | These are allowed to back Tailwind utilities and direct `var()` usage                                     |
| Runtime system tokens       | `:root` or `html` | `--glass-*`, `--dur-*`, `--z-*`, `--aurora-*`, `--mist-*`, `--grid-*`, `--noise-*`, panel highlight vars  | These are implementation tokens only; do not rely on Tailwind to generate utilities from them            |
| Compatibility aliases       | global utilities | `.dr-title-fantasy`, `.dr-heading`, `.dr-card--*`, `.dr-glow-ring`                                          | Keep working exactly as existing consumers expect                                                          |
| New additive alias          | global utility   | `.dr-font-display`                                                                                           | Allowed as the clean forward contract for explicit display moments; `.dr-title-fantasy` remains as an alias, not a removed API |

Locked ownership rule: do **not** keep `--glass-*`, `--dur-*`, or `--z-*` inside `@theme` after Phase 15 normalization. They are runtime variables, not utility namespaces.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                             |
| ----- | ----- | ------------------------------------------------- |
| xs    | 4px   | Icon gaps, inline padding, rune accents           |
| sm    | 8px   | Compact element spacing, chip spacing, input label gaps |
| md    | 16px  | Default element spacing, control padding, card internals |
| lg    | 24px  | Section padding, grouped blocks, sidebar clusters |
| xl    | 32px  | Layout gaps between major regions                 |
| 2xl   | 48px  | Major section breaks, auth card breathing room    |
| 3xl   | 64px  | Page-level vertical rhythm                        |

Exceptions: minimum 44px touch target for icon-only or compact interactive controls; decorative rune/diamond micro-elements may use 8px glyph sizing without forcing an 8px layout box.

## Typography

| Role    | Size                                 | Weight | Line Height |
| ------- | ------------------------------------ | ------ | ----------- |
| Body    | 16px                                 | 400    | 1.5         |
| Label   | 12px                                 | 600    | 1.4         |
| Heading | 20px                                 | 600    | 1.2         |
| Display | 28px base, clamp upward for hero titles | 600 | 1.1         |

Typography rules:

- Only two font weights are part of the system contract: `400` and `600`.
- Sora owns body, helper, and form text.
- JetBrains Mono owns uppercase labels, status/meta text, badges, and machine-like readouts.
- Cinzel is reserved for hero headings, page titles, and logo/brand moments only.
- Dense interface text, nav labels, form labels, and body copy must not switch to Cinzel.
- `.dr-heading` is the canonical fantasy heading contract.
- `.dr-font-display` is the forward-looking display alias; `.dr-title-fantasy` remains a compatibility alias.

## Color

| Role            | Value      | Usage                                                                    |
| --------------- | ---------- | ------------------------------------------------------------------------ |
| Dominant (60%)  | `#05070D`  | Foundation background, deepest canvas, app-wide dark base                |
| Secondary (30%) | `#22302C`  | Cards, sidebar, nav, elevated board panels, readable emerald/taupe surfaces |
| Accent (10%)    | `#00D97E`  | Primary action, active state emphasis, focus/glow moments only           |
| Destructive     | `#EF4444`  | Destructive actions only                                                 |

Accent reserved for: primary CTA fills, active navigation markers, focus borders, selected chips/toggles, emerald hover glows, key chart strokes, success/active status emphasis, and limited highlight text. Do not use emerald as a blanket border color for every panel.

Support colors:

- Taupe support: `#CFC3AE` for fantasy edge accents, secondary frames, dividers, and mist-warmed decorative surfaces.
- Gold support: `#C8943A` for premium hierarchy, title gradients, pending state, and high-value accents. Gold is not the default primary CTA fill.
- Mist support: `#D6E7DD` for light bloom, soft contrast on glass layers, and subtle highlight overlays.

Color behavior rules:

- The app must no longer feel like a flat black wall. Secondary surfaces should be visibly lighter than the dominant base.
- Use taupe/gold as support hierarchy, not as a competing accent system.
- Error and destructive messaging stay plain and readable, not theatrical.

## Surface And Atmosphere Contract

| Token / Element       | Prescriptive value             | Contract                                           |
| --------------------- | ------------------------------ | -------------------------------------------------- |
| `--glass-bg`          | `rgba(20, 28, 26, 0.76)`       | Default glass base for neutral cards               |
| `--glass-panel`       | `rgba(28, 38, 34, 0.82)`       | Lighter Emerald Guild panel for elevated surfaces  |
| `--glass-panel-soft`  | `rgba(35, 47, 43, 0.62)`       | Soft panel for quieter controls and secondary cards |
| `--glass-blur`        | `16px`                         | Standard glass blur                                |
| `--glass-blur-heavy`  | `24px`                         | Auth-grade or premium elevated blur                |
| `--glass-border-glow` | `rgba(0, 217, 126, 0.22)`      | Emerald hover/emphasis border glow                 |
| `--glass-border-gold` | `rgba(200, 148, 58, 0.24)`     | Gold premium border treatment                      |
| `--aurora-opacity`    | `0.12` max visual intensity    | Aurora stays atmospheric, never dominant over content |
| `--mist-opacity`      | `0.08`                         | Mist layer should soften the scene, not fog it     |
| `--grid-opacity`      | `0.16`                         | Hex-grid remains visible but secondary             |
| `--noise-opacity`     | `0.022`                        | Grain stays subtle and premium                     |
| Panel highlight top   | `rgba(255, 255, 255, 0.05)`    | Light catch for top edge of elevated glass         |
| Panel highlight bottom | `rgba(255, 255, 255, 0.014)`  | Depth cue for lower surface fade                   |

Atmosphere rules:

- Background language is `mist + subtle hex-grid + subtle noise`, not particles, sparks, or overt fantasy clutter.
- The hex-grid should read as a texture behind content, never as a competing foreground pattern.
- Glass panels should feel like lit quest-board surfaces: lighter than the base, still dark enough for readable contrast.
- Strong emerald glow belongs to important interactions and premium cards, not to every surface.

## Interaction And Motion Contract

| Element             | Contract                                                                 |
| ------------------- | ------------------------------------------------------------------------ |
| Base card hover     | Maximum lift `translateY(-2px)`; border brightens slightly; use `var(--shadow-elevated)` plus a restrained 1px emerald ring only |
| Emerald glow variant | `.dr-card--glow-emerald` is the only default strong emerald hover card treatment |
| Gold glow variant   | `.dr-card--glow-gold` is reserved for premium or reward-like emphasis    |
| Auth card           | `.dr-card--auth` may be visually heavier than base `.dr-card`, but remains additive and must not redefine the base contract |
| Primary button hover | Saturation lift plus controlled shadow increase; no bounce, no arcade flash |
| Entrance motion     | Use `--dur-reveal` and `--ease-out`; no system-layer motion longer than 800ms |
| Ambient motion      | Aurora drift only; no new looping effects added in this phase            |

Motion tokens:

- `--dur-instant`: `80ms`
- `--dur-fast`: `150ms`
- `--dur-normal`: `250ms`
- `--dur-slow`: `550ms`
- `--dur-reveal`: `800ms`
- `--ease-spring`: `cubic-bezier(0.22, 1, 0.36, 1)`
- `--ease-out`: `cubic-bezier(0.4, 0, 0.2, 1)`

Accessibility and performance rules:

- `prefers-reduced-motion: reduce` must disable aurora drift and CSS entrance animations.
- `will-change: transform` is allowed only on `.dr-aurora`, `.dr-aurora::before`, and `.dr-aurora::after`.
- Do not add `will-change` to cards, buttons, badges, or generic glow utilities.
- Phase 15 does not add JS-driven hover tracking, spotlight effects, or route-specific motion logic.

## Layering Contract

| Token        | Value | Ownership                              |
| ------------ | ----- | -------------------------------------- |
| `--z-bg`     | `0`   | Aurora and deepest atmospheric background |
| `--z-noise`  | `1`   | Noise/grain overlay                    |
| `--z-app`    | `2`   | Main app content wrapper               |
| `--z-sticky` | `10`  | Sticky headers, persistent rails, local sticky UI |
| `--z-modal`  | `50`  | Dialogs, overlays, modal surfaces      |
| `--z-toast`  | `200` | Toasts and urgent feedback layers      |

No route, component, or utility may introduce an ad-hoc magic z-index when one of these tokens already covers the case.

## Additive Compatibility Contract

These rules are locked for executor safety:

- Keep `.dr-card` as the neutral base card. Do not fold `.dr-card--auth`, `.dr-card--glow-emerald`, `.dr-card--glow-gold`, or `.dr-card--elevated` into the base class.
- Keep `.dr-btn-primary` and `.dr-btn-secondary` externally stable. Internal token consumption may change; external class names may not.
- Keep `.dr-heading` as the page-title/fantasy-heading contract.
- Keep `.dr-title-fantasy` working. It may become an alias, but it must not break current pages.
- Keep the global root mount structure from `app/layout.tsx`: `.dr-aurora`, `.dr-noise`, `.dr-app`.
- Do not introduce route-conditional background logic in this phase.
- Do not redesign `AppShell` markup, auth pages, or landing page markup in this phase.

## Copywriting Contract

| Element                  | Copy                                                                 |
| ------------------------ | -------------------------------------------------------------------- |
| Primary CTA              | Browse Raids                                                         |
| Empty state heading      | Nothing on the board yet                                             |
| Empty state body         | No active content matches this view yet. Clear filters or create a new raid to keep moving. |
| Error state              | We couldn't load this view. Refresh the page or try again in a moment. |
| Destructive confirmation | None in Phase 15. If a future phase adds one, use explicit action naming plus an irreversible warning. |

Tone rules:

- Use fantasy flavor in headings, section names, and empty states.
- Keep errors, helper text, and form guidance plain and readable.
- Prefer product-language clarity over RPG flourish when the user needs to recover from a problem.
- `Enter the Realm` remains reserved for auth hero CTA usage in later page phases, not as the default system-wide primary CTA label.
