  <files>app/hero/me/page.tsx</files>
  <action>
Rewrite `app/hero/me/page.tsx` as a `'use client'` component that fetches the authenticated user via `api.auth.getUser(token)`. The current file uses MOCK_HERO (a HeroProfile type) — the new file uses ApiUser instead.

Imports to use:
```typescript
'use client'

import { useEffect, useState } from 'react'
import { authStore } from '@/lib/auth-store'
import { api, type ApiUser } from '@/lib/api'
```

Do NOT import `MOCK_HERO`, `HexRadar`, `HeroProfile` type, or `preferred_roles`/`hard_skills`/`soft_profile`/`reputation` — these fields do not exist on `ApiUser` and were only in the mock.

State:
```typescript
const [user, setUser] = useState<ApiUser | null>(null)
const [loading, setLoading] = useState(true)
```

useEffect (depends on `[]` — run once on mount):
1. `const token = authStore.getToken()`
2. If no token: `setLoading(false)` and return (graceful fallback, no redirect — requirement says "doesn't crash", not "redirect")
3. If token: `setUser(authStore.getUser())` (optimistic cached display, may be null)
4. Call `api.auth.getUser(token)` — on resolve:
   - If `res.error`: `authStore.clear()`, `setUser(null)`
   - If `res.data`: `setUser(res.data)`
   - Either way: `setLoading(false)`

Render states:
- `loading === true` → return a loading placeholder: `<div className="p-6" style={{ color: 'var(--color-dr-muted)' }}>Loading profile…</div>`
- `user === null` → return sign-in prompt: `<div className="p-6" style={{ color: 'var(--color-dr-muted)' }}>Sign in to view your profile.</div>` (no crash, no redirect)
- `user !== null` → render profile using only `ApiUser` fields

Profile render (when user is not null) — use only fields that exist on `ApiUser`:
